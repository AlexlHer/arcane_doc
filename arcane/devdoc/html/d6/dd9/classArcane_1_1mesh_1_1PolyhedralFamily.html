Après appel à cette opération, il faut appeler endUpdate() pour notifier à l'instance la fin des modifications. Il est possible d'enchaîner plusieurs allocations avant d'appeler endUpdate(). Les unique_ids doivent être unique sur l'ensemble des sous-domaines. Il est possible de vérifier cela via checkUniqueIds(). items doit avoir le même nombre d'éléments que unique_ids et sera remplit en retour avec les numéros locaux des entités créées. Il est possible après création d'obtenir une vue sur les entités créés via view().

Int64UniqueArray my_unique_ids;
Int32UniqueArray my_local_ids;
my_unique_ids.add(25);
my_unique_ids.add(32);
family->addItems(my_unique_ids,my_local_ids);
family->endUpdate();
ENUMERATE_ITEM(iitem,family->view(my_local_ids)){
const Item& item = *iitem;
info() << "Item local_id=" << item.localId() << " unique_id=" << item.uniqueId();
}

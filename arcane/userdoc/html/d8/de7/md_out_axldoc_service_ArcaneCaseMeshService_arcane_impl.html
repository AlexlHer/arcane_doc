'ArcaneCaseMeshService' service

(implements Arcane::ICaseMeshService interface).

Service Arcane gérant un maillage du jeu de données.
Le maillage peut être soit lu à partir d'un fichier (élément 'filename'), soit généré dynamiquement (élément 'generator')

Summary of options

  • filename
  • generator
  • initialization (1 options)
    • variable (0 options)
  • nb-ghost-layer
  • partitioner
  • specific-reader

Detailed list of options

filename (string) 
<filename>value</filename>

Nom du fichier de maillage. S'il n'est pas définit il faut qu'un générateur de maillage soit spécifié.

generator (Arcane::IMeshBuilder) 
<generator name='value'>service configuration block</generator>

Possible values for tag name:
Cartesian3D
Cartesian2D
Sod3D
HoneyComb3D
HoneyComb2D
Service de création/lecture de maillage
Ce service doit être spécifié si aucun fichier de maillage n'est défini.

initialization

Informations d'initialisation. 
  Ces options permettent entre autre d'initialiser des variables sur des groupes
  avec des valeurs prédéfinies
</div>
initialization/variable [0... undefined]
Liste des variables à initialiser

nb-ghost-layer (int32=1) 
<nb-ghost-layer>value</nb-ghost-layer>

Nombre de couches de mailles fantômes à ajouter au maillage en cas de partitionnement.

partitioner (string=DefaultPartitioner) 
<partitioner>value</partitioner>

Nom du service de partitionnement à utiliser pour le découpage initial.
Si la valeur est 'External', alors on considère que le partitionnement a déjà été effectué en amont du lancement du code. Dans ce dernier cas, des fichiers de maillage déjà découpés doivent être disponibles.

specific-reader (Arcane::ICaseMeshReader) 
<specific-reader name='value'>service configuration block</specific-reader>

Service spécifique de lecture de maillage.
Ce service peut être spécifié si on souhaite éviter l'auto-détection du lecteur à utiliser ou si on souhaite ajouter des options spécifiques à un lecture donné.
